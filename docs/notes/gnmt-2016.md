# Reading Notes

The GNMT paper addresses three main issues in neural machine translation: training difficulty in deep RNN models, rare word (open vocabulary) problems, and coverage issues during decoding. Rather than proposing a completely new theoretical model, the paper focuses on engineering optimization and system-level refinement of the seq2seq architecture.

The model pipeline starts with wordpiece segmentation, which converts words into subword tokens to handle rare and unseen words. These tokens are then mapped into embedding vectors. The encoder consists of 8 layers: Layer 1 uses a bi-directional LSTM; Layers 2–8 use uni-directional LSTMs with residual connections. The decoder is also composed of 8 stacked LSTM layers with residual connections, and attention is applied using the top encoder layer's hidden states and the bottom decoder layer's state.

During inference, beam search (with beam size typically 8–12) is used instead of greedy decoding. This prevents locally optimal decisions from dominating the final output. Additionally, length normalization and coverage penalty are introduced to improve translation completeness and reduce short-sentence bias. GNMT represents a highly engineered version of RNN-based NMT, pushing the architecture close to its practical limits before the transition to Transformer models.

From this paper, I also learned the iterative design process of large-scale model development:

1. Validate the model structure using a small dataset.
2. Scale to the full dataset once stable.
3. Establish a strong baseline architecture.
4. Perform ablation studies to isolate the effect of each component.
5. Further polish and optimize the structure for production deployment.

# Learning From AI (GPT5.2)

### 1️⃣ 論文類型與歷史定位

GNMT is an engineering-oriented neural machine translation system built on deep LSTM + attention. It represents the peak optimization of RNN-based NMT before the Transformer era.

👉 This paper is not a fundamental innovation but a masterclass in system optimization and production engineering.

---

### 2️⃣ 核心目標

The paper aims to make neural machine translation production-ready by improving:

* Translation quality (BLEU + human evaluation)
* Long sentence handling
* Rare word translation
* Inference speed and deployment efficiency

👉 Engineering concerns drive architectural choices, not theoretical novelty.

---

### 3️⃣ 整體架構

Pipeline:

$$\text{Wordpiece tokens} \rightarrow \text{Embedding} \rightarrow \text{8-layer Encoder} \rightarrow \text{Attention} \rightarrow \text{8-layer Decoder} \rightarrow \text{Softmax} \rightarrow \text{Beam Search}$$

Key idea: Deep stacked LSTMs with residual connections.

👉 The architecture is a careful stacking of well-known components optimized for depth.

---

### 4️⃣ 深層 Encoder 設計

* Layer 1: Bi-directional LSTM
* Layers 2–8: Uni-directional LSTM
* Residual connections from layer 2 onward

Reason:

* Bi-layer gives full context
* Upper layers remain uni-directional to preserve parallelization

👉 Bidirectionality is needed only once; then feed forward for efficiency.

---

### 5️⃣ RNN 中的殘差連接

For layer $k \geq 2$:

$$h_t^k = \text{LSTM}(h_t^{k-1}) + h_t^{k-1}$$

Purpose:

* Improve gradient flow across layers
* Enable stable training of 8-layer RNN

Insight: Gradient now has an identity path.

👉 Residual connections are not just about depth; they are about trainability.

---

### 6️⃣ Attention 機制

Uses additive (Bahdanau-style) attention.

Important detail:

* Attention is computed from the bottom decoder layer
* It attends over all time steps of the top encoder layer

Attention allows dynamic access to source tokens at each decoding step.

👉 Attention links all encoder layers to the bottom decoder — enabling information bypass.

---

### 7️⃣ Wordpiece 分詞

Instead of word-level vocabulary, GNMT uses subword units.

Benefits:

* Handles rare and unseen words
* Reduces vocabulary size
* Improves copying of named entities

This solves open-vocabulary problems.

👉 Subword tokenization is a preprocessing insight that scales beyond NMT.

---

### 8️⃣ 推論中的 Beam Search

Decoding is done with beam search (typically K=8–12).

Why needed?

* Greedy decoding may choose locally optimal tokens
* Beam search approximates global sequence probability

Beam size trade-off:

* Larger K improves quality but increases latency

👉 Beam search is not just an inference detail; it affects data collection and training priorities.

---

### 9️⃣ 長度正規化與覆蓋懲罰

Final decoding score:

$$\text{score} = \frac{\log P(Y|X)}{\text{lp}(Y)} + \text{cp}(X, Y)$$

Length normalization:

* Prevents bias toward short sentences

Coverage penalty:

* Encourages attention to cover all source tokens
* Reduces under-translation

👉 Both penalties are heuristics addressing observed failure modes in inference.

---

### 🔟 工程化部署

GNMT includes:

* Model quantization for faster inference
* TPU deployment
* Pruning strategies in beam search
* Reinforcement learning fine-tuning

Key takeaway:

GNMT is not just a model innovation but a full production translation system design.

👉 Moving to production requires solving latency, memory, and batch processing challenges.
