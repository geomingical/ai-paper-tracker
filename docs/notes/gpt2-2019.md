# Reading Notes

The paper introduces GPT-2 (2019), which builds upon GPT-1. Unlike GPT-1, which was trained on relatively limited and curated datasets, GPT-2 is trained on a large-scale, diverse real-world corpus (WebText) using byte-level BPE tokenization.

Architecturally, GPT-2 largely follows the same Transformer decoder structure as GPT-1, with only minor modifications for training stability, such as changes in layer normalization placement and residual scaling (Section 2.3). The model is trained using the standard language modeling objective, maximizing p(next token | context) (Eq.(1)).

A key insight of GPT-2 is that many NLP tasks (e.g., question answering, translation) can be expressed as conditional sequence prediction problems. Since such input-output structures naturally exist in real-world text, the model implicitly learns these relationships during language modeling training. This enables GPT-2 to perform tasks in a zero-shot setting without task-specific supervision.

However, GPT-2 performs poorly on tasks like translation and summarization. This is not only due to limited data (e.g., only ~10MB of French text), but more importantly due to the lack of strong input-output alignment structures in the training data. In particular, translation requires cross-lingual alignment, and summarization requires information compression, both of which are not sufficiently represented in the dataset.

Therefore, the performance of GPT-2 on a task depends on whether the corresponding conditional structure (input → output) is sufficiently present in the training data. When such structures are abundant (e.g., QA), the model performs well; otherwise, its performance is limited.

# Learning From AI (GPT5.4)

### 1️⃣ 核心任務

GPT-2 的訓練目標是 language modeling：最大化 p(next token | context)，即預測下一個 token。

### 2️⃣ 架構本質

GPT-2 使用 decoder-only Transformer，架構基本延續 GPT-1，沒有本質改變。

### 3️⃣ 真正突破點

不是新模型，而是：

* 更大的模型（最高 1.5B parameters）
* 更大的資料（WebText）

### 4️⃣ 資料的重要性

WebText 來自 Reddit 高質量連結，包含大量自然語言任務示範（QA、翻譯片段等）。

### 5️⃣ 任務如何產生

任務不是被明確訓練，而是從資料中的 input → output 結構中學到。

### 6️⃣ Zero-shot 的本質

Zero-shot ≠ 無中生有
Zero-shot = 喚醒模型在訓練中已學到的條件結構

### 7️⃣ 任務成功條件

模型表現好 ⇔ 資料中存在足夠的 conditional structure
例如：

* QA：強（資料中很多）
* Translation：弱（幾乎沒有對齊資料）
* Summarization：更弱（缺乏壓縮結構）

### 8️⃣ 為什麼 QA 表現好

QA 多為 retrieval-like 任務，答案通常存在於 context 中，模型只需延續文本。

### 9️⃣ 為什麼 Summarization 表現差

Summarization 需要：

* 資訊壓縮
* 長距離理解
* 抽象能力

但 LM 只學 token 預測，沒有直接學這些能力。

### 🔟 最重要總結

GPT-2 的能力 ≈ 訓練資料中的條件機率結構
