# Reading Notes

The GPT-1 paper (2018) improves natural language understanding through a two-stage training framework: unsupervised pretraining and supervised fine-tuning.

In the pretraining stage, the authors train a language model on the BooksCorpus dataset (about 7,000 books) using a 12-layer Transformer decoder. The model is trained for 100 epochs with sequences of 512 tokens. During this stage, the model learns general language representations, including syntax and semantic relationships, by predicting the next token in a sequence.

In the fine-tuning stage, the pretrained model is adapted to specific NLP tasks such as classification, textual entailment, sentence similarity, and multiple-choice question answering. For each task, a linear classifier is added on top of the Transformer, and the whole model is fine-tuned using labeled datasets. Since these datasets are relatively small, the model can be trained efficiently using a batch size of 32 and about 3 epochs.

Overall, the computational cost of pretraining is much larger than fine-tuning, while fine-tuning allows the pretrained language representation to adapt to specific downstream tasks.

# Learning From AI (GPT)

### 1️⃣ Core Idea: Two-Stage Training

GPT-1 introduces a two-stage pipeline:

1. **Pretraining:** Train a language model on large unlabeled text.
2. **Fine-tuning:** Adapt the pretrained model to a specific supervised task.

👉 Two-stage training enables efficient transfer learning from unlabeled data to supervised tasks.

---

### 2️⃣ Language Model Objective

Pretraining uses a standard **next-token prediction objective**:

$$P(u_i \mid u_1, \ldots, u_{i-1})$$

The model learns statistical structure of language including syntax, semantics, and world knowledge.

👉 The language modeling objective implicitly captures linguistic structure without explicit supervision.

---

### 3️⃣ Architecture Choice

GPT-1 uses a **decoder-only Transformer** with masked self-attention.

Key components:

* Token embeddings
* Position embeddings
* Multi-head self-attention
* Feed-forward layers

This allows the model to capture long-range dependencies better than RNNs.

👉 Decoder-only architecture is simpler and more efficient for language generation than encoder-decoder designs.

---

### 4️⃣ Representation Learning

The pretrained language model produces contextual representations for tokens.

These hidden states contain information about:

* grammar
* semantic meaning
* context relationships

Fine-tuning mainly reuses these representations.

👉 Learned representations capture both syntactic and semantic knowledge useful for downstream tasks.

---

### 5️⃣ Fine-Tuning Mechanism

For each task, a simple **linear classifier** is added on top of the final Transformer layer.

Training objective:

$$\text{softmax}(h_m^l \cdot W_y)$$

All model parameters (including the Transformer) are updated during fine-tuning.

👉 Task-specific fine-tuning with minimal architectural changes enables rapid adaptation to new problems.

---

### 6️⃣ Minimal Architectural Changes

GPT-1 emphasizes **transfer with minimal changes**.

The only new parameters during fine-tuning are:

* task classifier weights
* delimiter/start token embeddings

The core Transformer remains the same across tasks.

👉 Minimal task-specific modifications keep the model general and reusable across different problem domains.

---

### 7️⃣ Task Input Transformation

Different NLP tasks are converted into **token sequences** so the same model can process them.

Examples:

Entailment:
$$\text{premise} \, \$ \, \text{hypothesis}$$

Similarity:
$$\text{sentence1} \, \$ \, \text{sentence2}$$
$$\text{sentence2} \, \$ \, \text{sentence1}$$

Question Answering:
$$\text{context} \, \text{question} \, \$ \, \text{answer}$$

👉 Universal task input transformation allows a single model to solve diverse NLP problems without architectural changes.

---

### 8️⃣ Delimiter Tokens

Delimiter tokens mark boundaries between different segments of text.

They are:

* treated like normal tokens
* assigned embeddings
* randomly initialized and trained during fine-tuning

They help the model distinguish structured inputs.

👉 Simple delimiter tokens enable explicit segmentation of multi-part inputs, serving as a minimal interface between task structure and the model.

---

### 9️⃣ Training Efficiency

Fine-tuning is fast because the model already understands language.

Typical setup:

* learning rate: 6.25e-5
* batch size: 32
* ~3 epochs sufficient

Pretraining does most of the heavy learning.

👉 Fast fine-tuning makes the pretrained model practical for rapid prototyping on new downstream tasks.

---

### 🔟 Experimental Insight

Experiments show that removing pretraining drastically reduces performance.

This demonstrates that:

**Pretraining learns general language representations that transfer to many tasks.**

👉 Ablation results confirm that unsupervised pretraining is the critical component driving the model's success.

---

### Key Takeaway

> Large-scale unsupervised language modeling → representation learning → task-specific fine-tuning. This paradigm later evolved into modern large language models.
