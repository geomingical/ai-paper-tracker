# Reading Notes

When a neural network becomes larger, its computational cost usually increases significantly. The MoE-2017 paper addresses this problem by introducing a sparsely gated Mixture-of-Experts layer, which allows the model to greatly increase its capacity without activating the entire network for every input. Instead of using one fully activated model, the MoE layer contains multiple expert networks and a trainable gating network.

When an input is received, the gating network calculates a score for each expert. The model then applies a top-$k$ selection mechanism to keep only a small number of experts and sets the rest to zero. The selected experts process the same input, and their outputs are combined through a weighted sum. These weights are dynamically determined by the gating network for each input, so different inputs may activate different experts and receive different combinations of expert outputs.

The novelty of MoE is that the routing rule is not manually designed. Instead, the gating network is trained together with the rest of the model through backpropagation. During training, the model learns how to distribute different inputs to different experts. This makes MoE different from a traditional rule-based if-condition system: it is a learned, input-dependent routing mechanism.

One advantage of this design is that the model can contain a very large number of parameters while each input uses only a small subset of them. Therefore, computational cost does not increase proportionally with the total number of parameters. The authors tested this idea mainly on language modeling and machine translation, but they also described the MoE layer as a general-purpose neural network component. This suggests that the same idea could potentially be applied to other prediction tasks, especially when the training dataset is very large.

# Learning From AI (GPT)

### 1️⃣ MoE 的核心不是「壓縮模型」，而是「稀疏啟動的大容量模型」

這篇和 distillation 的方向不同。Distillation 是把大模型知識壓到小模型；MoE 是把模型容量做得非常大，但每次輸入只啟動少數 experts，所以計算量不會跟參數量等比例暴增。

📍Abstract, Section 1.1, Section 1.2

### 2️⃣ Conditional Computation 是本文的主軸

傳統神經網路通常每個輸入都啟動整個模型；conditional computation 則是根據每個輸入，只啟動模型的一部分。MoE 是本文實作 conditional computation 的主要形式。

📍Section 1.1, Section 1.2

### 3️⃣ MoE Layer = Gating Network + 多個 Expert Networks + Weighted Sum

MoE layer 裡有很多 experts，每個 expert 是一個 feed-forward network；gating network 會為每個輸入產生 expert 權重，最後把 expert outputs 加權相加成 MoE output。

$$
y = \sum_i G(x)_i E_i(x)
$$

📍Section 2, Eq.(1)

### 4️⃣ Gating Network 是可訓練模型，不是人工規則

Gate 不是手寫規則，例如「地名送 expert 1」。本文的 gating network 有可訓練參數 $W_g$ 和 $W_{\text{noise}}$，會和整個模型一起用 backpropagation 訓練。

📍Section 2.1, Eq.(3)-(5), Training the Gating Network

### 5️⃣ Sparse Top-$k$ Gating 讓模型只計算少數 Experts

Gate 會先對所有 experts 算分數，再只保留 top-$k$，其餘設為 $-\infty$，softmax 後權重變成 0。

因此，若有 256 個 experts、$k=4$，每個輸入實際只計算 4 個 experts。

📍Section 2.1, Eq.(3)-(5), Appendix C.1

### 6️⃣ 加權相加是動態的，每個輸入都有自己的 Expert 組合

MoE output 是：

$$
y = \sum_i G(x)_i E_i(x)
$$

因為 $G(x)$ 取決於輸入 $x$，所以不同 token、不同位置、不同樣本可以啟動不同 experts，且加權比例也不同。

📍Section 2, Eq.(1), Section 1.2, Figure 1

### 7️⃣ Expert 的數量是 Hyperparameter，但 Expert 的分工是訓練學出來的

本文不是讓模型自己決定要長出幾個 experts；作者先設定 expert 數量，例如 32、256、4096、65536 等。

訓練中學出來的是：哪些輸入該送去哪幾個 experts，以及每個 expert 最後形成什麼樣的功能分工。

📍Appendix C.1, Section 5.2, Appendix D, Table 8

### 8️⃣ Load Balancing 是 MoE 成功的關鍵之一

如果沒有額外約束，gate 可能永遠偏好少數 experts，導致這些 experts 更常被訓練、變得更好、又更常被選，形成自我強化。

本文用 $L_{\text{importance}}$ 和 $L_{\text{load}}$ 來平衡 expert 的重要性與負載。

📍Section 4, Appendix A, Eq.(6)-(11), Table 6

### 9️⃣ MoE 的工程瓶頸不只是模型設計，還包含 Batch Size 與 Network Bandwidth

MoE 會讓每個 expert 收到的 batch 變小，造成 GPU 效率下降；分散式訓練時，跨裝置傳送 expert inputs/outputs 也可能受 network bandwidth 限制。

本文用 data parallelism 加 model parallelism，以及把 MoE convolutionally applied across time steps 來改善效率。

📍Section 3.1, Section 3.2

### 🔟 本文證明：參數量可大幅增加，但計算量不必等比例增加

在 1-Billion-Word language modeling、100B-word Google News corpus、machine translation 等實驗中，MoE 增加大量參數並改善 perplexity 或 BLEU，同時計算成本相對受控。

這是本文宣稱 conditional computation 第一次在深度網路中展現 major wins 的主要依據。

📍Section 5.1, Section 5.2, Section 5.3, Section 5.4, Conclusion, Tables 1-5, Table 8

## 一句話記憶

MoE 的核心思想是：用 gating network 對每個輸入動態選少數 experts，讓模型擁有巨大容量，但每次只付出小部分計算成本。

📍Abstract, Section 1.2, Section 2, Eq.(1), Section 2.1
