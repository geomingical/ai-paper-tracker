# Reading Notes

BERT was published in 2018. Its framework consists of two main stages: pre-training and fine-tuning. In the pre-training stage, one or two sentences are first converted into embeddings and then passed through a multi-layer bidirectional Transformer encoder. The model is trained on two tasks: Masked Language Modeling (MLM), which predicts masked words, and Next Sentence Prediction (NSP), which predicts whether the second sentence follows the first one in the original text.

依據：Introduction；Figure 1；Section 3.1。

Unlike GPT-1, which uses a left-to-right Transformer, BERT uses a bidirectional encoder. This means that each token can use both its left and right context when building its representation. After passing through the encoder, the final hidden state of each token is denoted as Tᵢ, while the final hidden state of [CLS] is denoted as C. The representation Tᵢ is used for masked token prediction, and C is used for sentence-level classification such as NSP.

Example input:
[CLS] the man went to the store [SEP] he bought a gallon of milk [SEP]

Example for MLM:
[CLS] the man went to [MASK] store [SEP] he bought a gallon of [MASK] milk [SEP]

Regarding the design of the model, the authors identified a critical limitation in GPT-1-style pre-training: it is unidirectional, so each token can only attend to the tokens on its left during training. To address this problem, they proposed a bidirectional pre-training strategy using MLM. They also introduced NSP to help the model learn relationships between sentence pairs. After showing that these design choices were effective through ablation studies, they further scaled the model up, leading to stronger performance.

# Learning From AI (GPT5.4)

### 1️⃣ 先不要把 BERT 當成「更大的 GPT」，而要把它當成「重新設計預訓練問題」

BERT 的關鍵不只是模型變大，而是作者先指出舊方法的結構性限制：單向 language model 很難學到深度雙向表示。

**設計啟示：**做 architecture 時，第一步不是先加參數，而是先問：現有方法到底卡在哪一種資訊流限制？是看不到右文脈？還是學不到句間關係？

---

### 2️⃣ Architecture 設計的起點，通常是「失效機制」而不是「流行模組」

BERT 的出發點是：如果每個 token 只能看左邊，那對需要左右文脈的任務會吃虧。

**設計啟示：**新模型不應該從「我也想加某個新模組」開始，而應該從「舊模型在什麼情況下失效」開始。先找瓶頸，再決定要改資訊路徑、loss、還是模型容量。

---

### 3️⃣ BERT 的核心骨架是 bidirectional Transformer encoder

BERT 沿用 Transformer encoder 當主幹，但把重點放在雙向上下文整合。也就是說，一個 token 的表示不是只看左邊，而是在每層都可同時結合左右資訊。

**設計啟示：**architecture 的第一層決策通常是資訊如何流動。像 BERT 這種例子，就是先改「資訊路徑」：從單向改成雙向。

---

### 4️⃣ 預訓練不是只有「餵資料」，而是「用任務塑造表示」

BERT 的表示不是單靠大量文本自動長出來的，而是被兩個預訓練任務共同塑形：MLM 與 NSP。

* MLM 逼模型學 token-level 上下文
* NSP 逼模型學 sentence-pair 關係

**設計啟示：**輸入資料決定模型看見什麼；訓練目標決定模型被逼著學什麼。設計 architecture 時，loss 和輸入格式要一起想。

---

### 5️⃣ 一筆 BERT 輸入其實是一個「統一包裝格式」

句對會被包成：

```md
[CLS] A [SEP] B [SEP]
```

這代表 BERT 不是分別處理 A 和 B，再事後比較；而是把整條 sequence 一起送入 encoder。

**設計啟示：**如果你希望模型學到兩個物件之間的關係，常見做法不是做兩套完全分開的表示，而是讓它們在同一計算圖中互相作用。

---

### 6️⃣ Token / segment / position embeddings 不是三個模型，而是同一個 token 的三種描述

每個 token 的輸入表示是三種 embedding 的和：

* token embedding：這是什麼詞
* segment embedding：它屬於 A 還是 B
* position embedding：它在序列哪個位置

**設計啟示：**當輸入有多種屬性時，不一定要拆成三條網路分開處理；有時更好的做法是先把它們整合成統一表示，再交給共享 backbone 處理。

---

### 7️⃣ MLM 的真正作用，不只是「補空格」，而是讓雙向表示可以被合法訓練

如果直接做普通雙向 language model，token 可能會間接看到自己，導致預測變得太 trivial。MLM 的設計就是為了避免這個問題。

**設計啟示：**很多看似奇怪的訓練技巧，其實是在修補一個更底層的學習漏洞。設計模型時，要問：這個 loss 是不是在避免資訊洩漏、捷徑學習或不合理的解法？

---

### 8️⃣ NSP 告訴你：同一個 encoder 可以同時承接不同層級的任務

BERT 用 token hidden states 做 MLM，也用 `[CLS]` 的整體表示做 NSP。這表示同一個 encoder 不只會做一種事，而是可以在共享表示上接不同任務頭。

**設計啟示：**不要把「一種模型只能一種功能」看得太死。更常見的情況是：

* backbone 負責抽通用表示
* task head 負責把表示讀成特定任務輸出

這是現代模型設計很重要的分層思維。

---

### 9️⃣ 好的 architecture 研究，不是一次改很多，而是做最小可驗證修改

BERT 論文很值得學的一點是：作者刻意讓很多設計接近 GPT，目的是讓比較更乾淨。這種做法能讓你比較知道性能提升到底來自哪個改動。

**設計啟示：**當你做新模型時，最怕同時改了 backbone、loss、資料處理、訓練設定、模型大小，最後根本不知道是哪個因素有效。研究上更好的習慣是：

1. 先提出最小改動版
2. 做 ablation
3. 確定核心機制有效
4. 再擴大模型

---

### 🔟 深度和廣度不是先後的教條，而是最後的 scaling 問題

BERT 給人的一個重要設計觀念是：先確認方向對，再談 scale。也就是說，先確認雙向表示和預訓練目標真的有效，再去討論 layer 數、hidden size、attention heads 要怎麼放大。

**設計啟示：**實務上可以用下面這個順序：

1. 先找失效機制
2. 先改資訊流與訓練目標
3. 做小模型驗證
4. 做消融
5. 最後才系統性比較加深、加寬、加頭數

所以問題通常不是「一定先加深還是先加寬」，而是：在你的假說已被驗證後，哪種 scaling 對目前瓶頸最敏感。

---

### 一頁總結

可以把 BERT 的設計邏輯濃縮成這句話：
> 先找出舊方法的資訊流限制，再用新的預訓練目標塑造表示，確認機制有效後，最後才擴模型容量。

這不只是 BERT 的讀法，也是一個很好的 architecture 設計思路。
