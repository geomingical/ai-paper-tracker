# Reading Notes

This paper presents memory optimization methods for GPUs during large-scale deep learning model training. Based on my understanding, memory consumption can be divided into fixed model states and dynamic residual states. The fixed states include model parameters (fp16, $2\Psi$), gradients (fp16, $2\Psi$), master weights (fp32, $4\Psi$), and optimizer states such as momentum (fp32, $4\Psi$) and variance (fp32, $4\Psi$), totaling $16\Psi$. For a 1.5B-parameter model, this alone requires about 24 GB of memory.

The dynamic states consist of residual memory, primarily activations that scale with batch size. Consequently, the major gap in training giant models is that the required GPU memory is several times larger than the model size itself, making the system strictly memory-bound.

Traditional distributed training relies on three major strategies. Data parallelism (DP) replicates the entire model on every GPU, which is highly memory-inefficient. Model parallelism (MP) distributes parameters within each layer across different GPUs, but suffers from high communication overhead. Pipeline parallelism (PP) distributes parameters sequentially by layers across GPUs, but its pipeline nature introduces bubbles that leave many GPUs idle.

To resolve these bottlenecks, ZeRO partitions the fixed model states across different GPUs and offloads dynamic states, such as activation checkpoints, to CPU memory when necessary. When these dynamic states are required for training, they are fetched back to the GPU on the fly. Crucially, in multi-server environments, CPU-GPU data transfers are confined within the same server to avoid slow PCIe bottlenecks. Cross-server communication is handled through high-speed GPU-to-GPU networks because bandwidth is the limiting resource.

# Learning From AI (Gemini 3.5 Flash)

### 1️⃣ 記憶體的物理壁壘：Memory Wall

隨著模型規模邁向百億甚至兆級參數，單一 GPU 的顯存容量會迅速被榨乾。

傳統的資料並行 (DP) 因為在每張卡上完整複製模型，導致記憶體效率極差，面臨嚴重的冗餘瓶頸。

### 2️⃣ $16\Psi$ 的靜態黑洞

在 Adam 優化器與 mixed-precision training 下，儲存 model states 需要高達 $16\Psi$ bytes 的顯存空間，其中 $\Psi$ 為參數總數。

這其中，最佔空間的不是參數本體 ($2\Psi$) 或梯度 ($2\Psi$)，而是高達 $12\Psi$ 的 optimizer states，包含 fp32 參數備份、momentum 與 variance。

### 3️⃣ 顯存與記憶體的「工作檯 / 置物櫃」分工

GPU 顯存 (VRAM) 是極速但容量有限的本地工作檯，GPU cores 只能直接讀取這裡的資料。

系統記憶體 (CPU RAM) 則是容量巨大但受限於 PCIe 傳輸頻寬的遠端置物櫃。ZeRO 透過動態搬移，讓這兩種記憶體互補。

### 4️⃣ ZeRO 的靈魂哲學：動態借用

ZeRO 的核心思想是打破靜態複製，實施動態調度。

它採取「記憶體用切片的，計算時動態借用」的策略，把模型狀態平均拆分，只在特定計算時刻才在網路間互通有無，算完隨即釋放。

### 5️⃣ ZeRO-DP Stage 1：優化器狀態切片 $P_{os}$

每張 GPU 固定只儲存與更新屬於自己那 $\frac{1}{N_d}$ 的 optimizer states，其中 $N_d$ 為 GPU 總數。

這個階段能為 model states 帶來 4 倍的記憶體削減，且更新權重後的 All-Gather 總通訊量與傳統 DP 相同。

### 6️⃣ ZeRO-DP Stage 2：梯度切片 $P_{os+g}$

在反向傳播時，某一層的梯度一旦算出來，會立刻透過 Reduce-Scatter 傳送給負責的 GPU，並在其他卡上釋放空間。

疊加 Stage 1 後，model states memory 總共能省下約 8 倍，且通訊量依然維持不變。

### 7️⃣ ZeRO-DP Stage 3：參數切片 $P_{os+g+p}$

Stage 3 連模型參數本體也拆散存放，在前向與反向傳播時採取「算到哪、借到哪、算完立刻釋放」的極致排程。

記憶體削減幅度與 GPU 數量成線性正比，約為：

$$
\frac{1}{N_d}
$$

這讓訓練任意大小的模型變得可能，代價是總通訊量溫和地增加約 50%。

### 8️⃣ ZeRO-R 掃除動態脂肪：Residual States

除了固定權重，ZeRO-R 專門清理動態產生的 residual states。

它包含 activation partitioning ($P_a$) 與 CPU offloading ($P_{a+cpu}$)、將臨時緩衝區鎖定為固定大小的設計 ($C_B$)，以及防止 OOM 崩潰的線上記憶體去碎片化 ($M_D$)。

### 9️⃣ 反直覺的「超線性加速」現象

隨著投入的 GPU 數量增加，每張卡承擔的記憶體開銷會變少，進而允許每張卡塞入更大的 local batch size。

Batch size 變大會提升運算強度，讓 GPU cores 維持在較高效率，通訊開銷也被稀釋，使得整體訓練速度的增長可能超越單純的線性加倍。

### 🔟 大模型訓練的「民主化」與易用性

傳統的 model parallelism (MP) 與 pipeline parallelism (PP) 需要工程師大幅重構模型架構程式碼，並面臨收斂與架構功能上的限制。

ZeRO 介面相容於 PyTorch 標準模組，科學家不需要修改模型程式碼，就能用接近資料並行的方式訓練百億甚至兆級參數模型。

## 一句話記憶

ZeRO 的核心不是讓模型變小，而是把 optimizer states、gradients、parameters 和 residual states 分散或動態搬移，讓巨大模型在有限 GPU memory 下可以被訓練。
